# Client Onboarding Guide

This guide walks a developer through setting up a new podcast client in the
production pipeline. After completing these steps, the client's episodes will
be processed automatically: transcribed, censored, clipped, mastered, and
distributed to YouTube, Spotify, Twitter, and other platforms without manual
intervention.

---

## Overview

The pipeline takes one command (`uv run main.py --client <slug> latest`) and
produces:

- Transcription with word-level timestamps
- AI-selected highlight clips (3 by default) with burned-in subtitles
- LUFS-normalized master MP3 with chapter markers
- YouTube full-episode video + YouTube Shorts (vertical clips)
- Twitter thread with clip links and hashtags
- SEO-optimized blog post from the transcript
- Updated RSS feed (Podcasting 2.0 compliant, compatible with Spotify)
- Thumbnail (1280x720) with episode title overlay
- Discord notifications at key pipeline milestones
- Full-text search index of all episodes

Everything is genre-aware: comedy shows sound different from true crime shows,
which sound different from business interview shows. The AI writing (clips,
blog, social posts) uses a `voice_persona` you configure per client.

---

## Prerequisites

Before onboarding a new client, confirm the following are installed on the
production machine:

| Requirement | Version | Notes |
|-------------|---------|-------|
| Python | 3.12+ | Check with `python --version` |
| uv | Latest | Package manager — install from https://docs.astral.sh/uv/ |
| FFmpeg | Any recent | Must be at `C:\ffmpeg\bin\ffmpeg.exe` (Windows) or on PATH |
| NVIDIA GPU + CUDA | Optional | Whisper transcription is 3-7x faster with GPU; CPU fallback works |
| OpenAI API key | Required | GPT-4o for content analysis — set as `OPENAI_API_KEY` env var |

Platform credentials (YouTube, Twitter, etc.) are configured per-client in
the YAML file. You do not need all of them — the pipeline skips platforms
whose credentials are missing.

## Information to Collect from Client

Gather these fields before starting setup. Fill them into the client YAML
(see `clients/client-template.yaml` for the annotated template).

### Identity

| Field | Required? | Description | Example |
|-------|-----------|-------------|---------|
| `client_name` | Yes | Unique identifier, lowercase with hyphens. Used as directory and CLI flag. | `honey-and-hustle` |
| `podcast_name` | Yes | Display name of the podcast, used in RSS feed and AI-generated content. | `Honey & Hustle` |

### Episode Source

| Field | Required? | Description | Example |
|-------|-----------|-------------|---------|
| `episode_source` | Yes | Where to download audio from. Options: `dropbox`, `rss`, `local`. | `rss` |
| `rss_source.feed_url` | If source=rss | Public RSS feed URL. Must be a real, reachable feed. | `https://feeds.captivate.fm/honeyandhustle/` |
| `rss_source.episode_index` | If source=rss | Which episode to process: 0=latest, 1=second newest, etc. | `0` |

### Dropbox (only if episode_source is "dropbox")

| Field | Required? | Description | Example |
|-------|-----------|-------------|---------|
| `dropbox.folder_path` | If source=dropbox | Dropbox path to raw audio files | `/My Podcast/raw_files` |
| `dropbox.finished_folder` | If source=dropbox | Dropbox path where finished MP3s are uploaded | `/My Podcast/finished_files` |
| `dropbox.edited_folder` | If source=dropbox | Dropbox path for edited source audio | `/My Podcast/edited_files` |
| `dropbox.app_key` | If source=dropbox | Dropbox app key — can use `DROPBOX_APP_KEY` env var instead | `abc123...` |
| `dropbox.app_secret` | If source=dropbox | Dropbox app secret — can use `DROPBOX_APP_SECRET` env var instead | `xyz789...` |
| `dropbox.refresh_token` | If source=dropbox | Dropbox OAuth2 refresh token — can use `DROPBOX_REFRESH_TOKEN` env var instead | `sl.long-token...` |

### Content Settings

| Field | Required? | Description | Example |
|-------|-----------|-------------|---------|
| `content.num_clips` | Optional | Number of highlight clips to extract per episode. Default: 3 | `3` |
| `content.clip_min_duration` | Optional | Minimum clip length in seconds. Default: 15 | `15` |
| `content.clip_max_duration` | Optional | Maximum clip length in seconds. Default: 30 | `60` |
| `content.whisper_model` | Optional | Whisper model size: `tiny`, `base`, `small`, `medium`, `large`. Default: `medium`. Use `small` for accents that need tuning. | `small` |
| `content.clip_selection_mode` | Optional | How clips are chosen. `content` = AI picks for insight/narrative. Default (omitted) = energy-based (RMS peaks). Use `content` for interview/true crime genres. | `content` |
| `content.compliance_style` | Optional | How aggressively to flag YouTube guideline violations. `strict` = flag conservatively (true crime). `standard` = normal shows. Default (omitted) = permissive (comedy). | `strict` |
| `content.names_to_remove` | Optional | Host first names or full names to audio-censor. Case-insensitive match in transcript. | `["Jane", "John Smith"]` |
| `content.words_to_censor` | Optional | Slurs or profanity to audio-censor. Case-insensitive. | `["badword", "slur"]` |
| `content.voice_persona` | Optional | Multi-line prompt telling GPT-4o how to write for this show. Covers tone, audience, style dos/don'ts. If null, uses the default comedy voice. | See genre guides below |
| `content.blog_voice` | Optional | Multi-line prompt for blog post writing style. Include BAD/GOOD examples for contrast. | See genre guides below |
| `content.scoring_profile` | Optional | Custom topic scoring criteria for the topic engine. If omitted, uses comedy defaults. See genre guides below for structure. | See genre guides below |

## Setup Steps

Follow these steps in order.

### 1. Scaffold the client directory

```bash
uv run main.py init-client <slug>
```

This creates `clients/<slug>/` with subdirectories for credentials and output
isolation. Replace `<slug>` with the `client_name` value (e.g., `honey-and-hustle`).

### 2. Create the client YAML config

```bash
cp clients/client-template.yaml clients/<slug>.yaml
```

Open `clients/<slug>.yaml` in a text editor and fill in all required fields.
Use `clients/client-template.yaml` as the annotated reference — every field
has an inline comment explaining its purpose, valid values, and defaults.

### 3. Fill in required fields

At minimum, set:
- `client_name` (must match the filename slug)
- `podcast_name`
- `episode_source` (`dropbox`, `rss`, or `local`)
- Source-specific fields (RSS feed URL or Dropbox folder paths)
- At least one platform credential section (YouTube, Twitter, etc.)
- `content.voice_persona` (strongly recommended — affects all AI output quality)

### 4. Set platform credentials

Credentials can go in the YAML (under the platform section as shown) **or**
as environment variables. Environment variables take precedence if both are set.

For YouTube, you must run the OAuth setup flow:

```bash
uv run main.py setup-client <slug> youtube
```

This opens a browser window for the client to authorize their Google account.
The token is saved to the path specified in `youtube.token_pickle`.

### 5. Validate the config

```bash
uv run main.py validate-client <slug>
```

This checks that all required fields are present and correctly typed. Fix any
errors reported before proceeding.

### 6. Ping external connections

```bash
uv run main.py validate-client <slug> --ping
```

This makes live test calls to configured external services (Dropbox, RSS feed,
Discord) and reports which ones are reachable. Failures here indicate missing
credentials or network issues — not config syntax errors.

### 7. Dry-run the full pipeline

```bash
uv run main.py --client <slug> latest --dry-run
```

This validates the entire pipeline wiring without downloading audio, calling
external APIs, or writing any files. It catches import errors, missing modules,
and config gaps before you run on real audio.

### 8. Process the first real episode

```bash
uv run main.py --client <slug> latest
```

The pipeline will:
1. Download the latest episode audio
2. Transcribe with Whisper
3. Analyze content (censorship, clips, chapters, social posts)
4. Apply censorship and normalization
5. Extract clips, generate subtitles, create videos and thumbnails
6. Upload to all configured platforms
7. Generate blog post and update RSS feed
8. Send Discord notifications at each major milestone

Processing time: 20-60 minutes depending on episode length and GPU availability.

### 9. Verify output

Check the `output/<slug>/` directory for all generated files. Review the
Discord notifications (if configured) for any upload errors. Run
`uv run main.py list-clients` to confirm the client appears in the registry.

## Genre Quick-Start Guides

Copy the recommended settings for each genre as a starting point.

### Comedy

Reference config: none (comedy is the default pipeline behavior)

```yaml
content:
  clip_selection_mode: null   # omit — defaults to energy-based (RMS peaks)
  compliance_style: null      # omit — defaults to permissive (allows profanity, dark humor)
  voice_persona: |
    You write for [Show Name] — an edgy comedy podcast. The hosts are [Host Names].
    The show's humor is dark, irreverent, and self-aware. The audience loves absurdist
    takes and is not easily offended. Do not sanitize the content.
    Write captions and clips that match the show's voice, not generic podcast language.
  blog_voice: |
    Write in the show's edgy, comedic voice.
    BAD: "In this episode, the hosts explore themes of..."
    GOOD: "The guys went full chaos this week. Here's what happened."
```

### True Crime

Reference config: `clients/true-crime-client.yaml`

```yaml
content:
  whisper_model: "small"          # Better for non-American accents
  clip_selection_mode: "content"  # Flat audio — pick clips for narrative, not volume
  compliance_style: "strict"      # Conservative flagging for real crime content
  voice_persona: |
    You write for [Show Name] — a serious true crime podcast examining unsolved cases.
    Your output is measured, respectful to victims and families, and evidence-focused.
    Avoid sensationalism. Write like a journalist, not an entertainer.
  blog_voice: |
    Write in a clear, investigative tone.
    BAD: "A chilling case that will keep you up at night..."
    GOOD: "This episode examines the 1987 disappearance of [Name], a case
    that went cold for 30 years before new DNA evidence emerged in 2023."
```

### Business / Interview

Reference config: `clients/business-interview-client.yaml`

```yaml
content:
  clip_selection_mode: "content"  # Pick clips for insight, not energy peaks
  compliance_style: "standard"    # Normal YouTube guidelines
  voice_persona: |
    You write for [Show Name] — a business interview podcast. Output is direct,
    substantive, and professional without being corporate. Highlight concrete lessons,
    hard decisions, and numbers. Never use hype language.
  blog_voice: |
    Write for a professional audience interested in entrepreneurship.
    BAD: "In this inspiring episode, our guest shares their amazing journey..."
    GOOD: "[Guest] built a $4M bootstrapped business in 18 months. Here's what
    she learned about pricing, retention, and when to say no to investors."
```

## Monitoring Setup

The pipeline sends Discord notifications at key milestones (transcription
complete, clips approved, uploads finished, any errors).

To enable Discord notifications:

1. Create an incoming webhook in Discord (Server Settings > Integrations > Webhooks)
2. Copy the webhook URL
3. Set it in the client YAML:
   ```yaml
   discord:
     webhook_url: "https://discord.com/api/webhooks/123456789/your-token-here"
   ```
   Or set the `DISCORD_WEBHOOK_URL` environment variable.

Notifications include: episode title, pipeline step completed, upload URLs
for each platform, and error details if any step fails.

## Troubleshooting

**FFmpeg not found**

The pipeline fails immediately if FFmpeg is missing.
- Windows: Install to `C:\ffmpeg\` and confirm `C:\ffmpeg\bin\ffmpeg.exe` exists.
- Linux/Mac: `brew install ffmpeg` or `apt install ffmpeg`, then confirm `ffmpeg` is on PATH.
- Set the `FFMPEG_PATH` env var to override the default search path.

**Missing API keys / credentials**

`validate-client` reports which required fields are missing. The most common:
- `OPENAI_API_KEY` env var — required for transcription analysis and content generation
- Dropbox credentials — required only when `episode_source: "dropbox"`
- YouTube credentials — run `uv run main.py setup-client <slug> youtube` to generate the token

**RSS feed not found / unable to parse**

- Confirm the feed URL is publicly accessible (open it in a browser)
- Some feeds require a user-agent header — try the URL with `curl -A "Mozilla/5.0" <url>`
- feedparser handles most RSS/Atom variants; if it still fails, the feed may use a non-standard format

**validate-client failures**

Common causes:
- `client_name` in the YAML does not match the filename (e.g., YAML says `my-podcast` but file is `mypodcast.yaml`)
- Required fields are set to `null` instead of a real value
- `episode_source: "rss"` without a corresponding `rss_source.feed_url`
- Invalid enum value (e.g., `compliance_style: "lenient"` — valid values are `strict` or `standard`)

**Pipeline hangs during transcription**

- GPU transcription uses CUDA — if the GPU is busy, Whisper queues. Check GPU usage with `nvidia-smi`.
- If CPU fallback is active, a 60-minute episode takes 30+ minutes. This is normal.
- If the process exits with a CUDA out-of-memory error, try a smaller whisper_model: `small` or `base`.

**YouTube upload fails with "quotaExceeded"**

YouTube Data API v3 has a 10,000 unit/day quota. Full episode + 3 Shorts uses ~1,600 units.
- Check quota at https://console.cloud.google.com/apis/dashboard
- Quota resets at midnight Pacific time
- Use `--auto-approve` to skip the interactive clip approval step and reduce API calls

**First run produces no clips**

- Check `output/<slug>/` for the analysis JSON — it shows what clips were selected
- If `clip_selection_mode` is not set and the audio is flat (interview format), switch to `"content"` mode
- Increase `clip_max_duration` if the episode has long, slow-building moments
