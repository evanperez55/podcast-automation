**TestAudioDucking (new class — AUDIO-01):**

Keep the existing `TestAudioProcessor`, `TestCensoringTimestampAccuracy`, `TestClipDurationValidation`, and `TestNormalizeAudio` (to be replaced below) classes intact — add `TestAudioDucking` as a NEW class.

Each test in `TestAudioDucking` patches `audio_processor.AudioSegment.from_file` and calls `apply_censorship()` with `start_seconds`/`end_seconds` timestamps. Tests verify the ducking behavior:

- `test_duck_no_beep_sound_used`: after applying censorship, the mock for `self.beep_sound` (set to a Mock on the processor) should NOT have `__getitem__` or `__mul__` called — ducking must NOT use `self.beep_sound` at all
- `test_duck_applies_gain_reduction`: the mock AudioSegment returned by slicing must have `apply_gain` called with a negative value (<= -30 dB) — verifies volume is reduced
- `test_duck_applies_fade_in_and_out`: the apply_gain result must have `fade_in()` and `fade_out()` both called — verifies smooth edges
- `test_duck_short_segment_no_fade_overrun`: for a 100ms segment (start=10.0, end=10.1), fade_in and fade_out must still be called (pydub handles short segments gracefully, but the cap logic must not skip fades entirely)
- `test_duck_preserves_audio_before_and_after`: the final audio should be built from `audio[:start_ms] + ducked + audio[end_ms:]` — verify `__add__` is called twice (once for each + operator)

For all TestAudioDucking tests, set `audio_processor.beep_sound = Mock(spec=AudioSegment)` on the processor instance to ensure we can assert it is NOT called.

**TestNormalizeAudio (full rewrite — AUDIO-02, AUDIO-03):**

Replace ALL existing TestNormalizeAudio test methods. The new tests use `@patch("subprocess.run")` instead of patching AudioSegment. They also patch `audio_processor.AudioProcessor._parse_loudnorm_json` where needed to control parsed output.

- `test_normalize_raises_on_missing_file`: FileNotFoundError raised when input path does not exist (keep this behavior — no subprocess involvement)
- `test_normalize_calls_ffmpeg_twice`: `subprocess.run` called exactly twice (two-pass loudnorm). Verify first call args contain `loudnorm=I=-16` and `print_format=json`. Verify second call args contain `linear=true` and `measured_I`.
- `test_normalize_uses_lufs_target_from_config`: The FFmpeg loudnorm filter string in call 1 uses `Config.LUFS_TARGET` value (e.g., `f"I={Config.LUFS_TARGET}"`). Use `@patch.object(Config, "LUFS_TARGET", -16)`.
- `test_normalize_warns_on_agc_fallback`: When `_parse_loudnorm_json` returns `{"normalization_type": "dynamic", "input_i": "-14.0", "input_tp": "-1.0", "input_lra": "5.0", "input_thresh": "-24.0", "target_offset": "0.0"}`, a `WARNING` log message is emitted (not an exception). Use `pytest.warns` or `caplog` to verify the warning.
- `test_normalize_logs_input_output_gain_lra`: When first pass returns `input_i=-23.0` and second pass returns `output_i=-16.0, output_lra=8.0`, an INFO log message is emitted containing all four values. Use `caplog` fixture (`caplog.set_level(logging.INFO)`).
- `test_normalize_returns_output_path`: `normalize_audio(audio_path, output_path)` returns `output_path`.
- `test_normalize_passes_devnull_to_subprocess`: `subprocess.run` is called with `stdin=subprocess.DEVNULL` (prevent stdin hang per pitfall 5 in RESEARCH.md).

For subprocess mocking: mock `subprocess.run` to return a `Mock(returncode=0, stderr="...\n{...json...}\n")`. Use the actual JSON strings from RESEARCH.md first-pass/second-pass examples so `_parse_loudnorm_json` can be tested end-to-end in some tests, or patch `_parse_loudnorm_json` directly in tests that only care about downstream behavior.
