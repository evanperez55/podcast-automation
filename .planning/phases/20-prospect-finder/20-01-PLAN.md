<feature>
  <name>ProspectFinder — iTunes search, RSS enrichment, YAML scaffolding</name>
  <files>prospect_finder.py, tests/test_prospect_finder.py</files>
  <behavior>
    DISC-01 — iTunes Search:
    - search() calls requests.get to itunes.apple.com/search with term, media=podcast, genreId, limit=200
    - search() filters results client-side by min_episodes <= trackCount <= max_episodes
    - search() returns at most `limit` results
    - search() returns empty list on API error (logs warning, does not raise)
    - search() with genre_id=None omits genreId param

    DISC-02 — RSS Enrichment:
    - enrich_from_rss() extracts email from itunes_owner.email first
    - enrich_from_rss() falls back to itunes_email, then author_detail.email
    - enrich_from_rss() extracts twitter and instagram links via regex on description
    - enrich_from_rss() returns last_pub_date as ISO date string from first entry
    - enrich_from_rss() returns episode_count from len(feed.entries)
    - enrich_from_rss() handles bozo feeds gracefully (log warning, continue parsing)
    - enrich_from_rss() returns empty dict on total parse failure

    DISC-03 — Save Prospect:
    - save_prospect() calls init_client(slug) if YAML does not exist
    - save_prospect() writes prospect: block with itunes_id, feed_url, genre, episode_count, contact_email, website, social_links
    - save_prospect() pre-fills content.voice_persona and content.compliance_style from GENRE_DEFAULTS when genre_key provided
    - save_prospect() sets episode_source=rss and rss_source.feed_url
    - save_prospect() registers prospect in OutreachTracker via add_prospect() at "identified" status
    - save_prospect() returns Path to the YAML file
    - save_prospect() is idempotent — calling twice with same slug does not error (init_client no-ops, tracker INSERT OR IGNORE)

    Module constants:
    - GENRE_IDS dict: comedy=1303, true-crime=1488, business=1321, technology=1318, society=1324
    - GENRE_DEFAULTS dict: comedy, true-crime, business with voice_persona, compliance_style, categories
    - _genre_key_from_name() maps iTunes primaryGenreName to GENRE_DEFAULTS key
  </behavior>
  <implementation>

    - Import requests, feedparser, re, yaml, pathlib.Path, datetime
    - Import Config from config, logger from logger, init_client from client_config, OutreachTracker from outreach_tracker
    - self.enabled = True (no credentials required for iTunes)
    - GENRE_IDS and GENRE_DEFAULTS as module-level constants (see RESEARCH.md Pattern 3 and Code Examples section for exact values)
    - search() per RESEARCH.md Pattern 1: requests.get with timeout=10, raise_for_status, client-side trackCount filter

    - _genre_key_from_name() lowercases, replaces spaces with hyphens, strips "and" — maps "True Crime" to "true-crime", "Comedy" to "comedy"
