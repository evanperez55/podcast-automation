3. **Information to Collect from Client** — A checklist table with columns: Field | Required? | Description | Example. Include EVERY field from the client YAML:
   - Show name, client slug (lowercase-hyphenated)
   - Episode source preference (dropbox/rss/local) and RSS feed URL if applicable
   - Dropbox folder paths (if dropbox source)
   - Platform credentials: YouTube OAuth, Twitter API keys, Instagram token, TikTok keys, Discord webhook URL
   - Content preferences: number of clips, clip duration range, names to censor, words to censor
   - Voice persona description (how AI should write about their show)
   - Blog voice description
   - Scoring profile (genre-specific topic scoring criteria)
   - Clip selection mode (content vs energy-based)
   - Compliance style (strict/standard/default)
   - RSS metadata: description, author, email, website URL, artwork URL, language, explicit flag, iTunes categories
   - Branding: logo path

4. **Setup Steps** — Numbered walkthrough:
   a. Run `uv run main.py init-client <slug>` to scaffold directory
   b. Copy `clients/client-template.yaml` to `clients/<slug>.yaml`
   c. Fill in required fields (reference the checklist)
   d. Add platform credentials (either in YAML or as env vars)
   e. Run `uv run main.py validate-client <slug>` to check config
   f. Run `uv run main.py validate-client <slug> --ping` to verify external connections
   g. Run `uv run main.py --client <slug> latest --dry-run` to validate full pipeline
   h. Run `uv run main.py --client <slug> latest` for first real episode

7. **Troubleshooting** — Common issues: missing FFmpeg, missing API keys, RSS feed not found, validate-client failures

The template must include inline comments (# lines) explaining EVERY field. Structure:

```yaml
# ============================================================
# Client Configuration Template
# ============================================================
# Copy this file to clients/<your-client-slug>.yaml
# Fill in required fields, leave optional ones as null or remove them
# Validate with: uv run main.py validate-client <your-client-slug>
# Full docs: see ONBOARDING.md in project root
# ============================================================

# REQUIRED: Unique identifier, lowercase with hyphens (e.g., "my-podcast")
client_name: "your-client-slug"

# REQUIRED: Display name of the podcast
podcast_name: "Your Podcast Name"
```

For EVERY field include:
- Whether it is REQUIRED or OPTIONAL
- What it does in plain English
- Valid values for enum-like fields (episode_source: dropbox|rss|local, compliance_style: strict|standard, clip_selection_mode: content|energy)
- Default behavior when null/omitted (e.g., "Falls back to DROPBOX_APP_KEY env var")
- Example value where helpful

Include ALL fields from the union of example-client.yaml, honey-and-hustle.yaml, true-crime-client.yaml, and business-interview-client.yaml:
- client_name, podcast_name
- branding.logo_path
- episode_source, rss_source (feed_url, episode_index)
- dropbox section (folder_path, finished_folder, edited_folder, app_key, app_secret, refresh_token)
- youtube section (client_id, client_secret, token_pickle)
- twitter section (api_key, api_secret, access_token, access_secret)
- instagram section (access_token, account_id)
- tiktok section (client_key, client_secret, access_token)
- discord section (webhook_url)
