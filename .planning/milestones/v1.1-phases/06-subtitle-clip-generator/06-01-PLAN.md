    ```python
    class SubtitleClipGenerator:
        def __init__(self):
            self.enabled = os.getenv("USE_SUBTITLE_CLIPS", "true").lower() == "true"
            self.ffmpeg_path = Config.FFMPEG_PATH
            self.logo_path = Config.ASSETS_DIR / "podcast_logo.png"

            self.bg_color = os.getenv("SUBTITLE_BG_COLOR", "0x1a1a2e")
            self.fonts_dir = str(Config.ASSETS_DIR / "fonts")
    ```

    Key implementation details (from RESEARCH.md):
    - `normalize_word_timestamps(words)`: Module-level function. Three passes: (1) interpolate missing timestamps, (2) close gaps < 150ms, (3) resolve overlaps. See Pattern 3 in research.
    - `_escape_ffmpeg_filter_path(path)`: Static method. Backslash to forward slash, then escape drive colon. See Pattern 5.
    - `_to_bgr_hex(rgb_hex)`: Static method. Convert `0xRRGGBB` or `#RRGGBB` to `BBGGRR` string for ASS color tags.
    - `_group_into_cards(words, max_words=3)`: Module-level function. Simple chunking by max_words.
    - `_generate_ass_file(self, clip_words, output_path, width, height)`: Creates ASS via pysubs2. Define "Default" SSAStyle with Anton font, fontsize=72, bold, white primary, black outline=3, shadow=1, alignment=2 (bottom-center), marginv=80. Group words into cards of max 3. For each card, create one SSAEvent per word where that word is the active (highlighted) word. Active word gets `{\c&HBBGGRR&}` accent color prefix and `{\c&HFFFFFF&}` reset suffix. Surrounding words in the card are plain white. ALL word text must be `.upper()` and have `{` and `}` escaped as `\{` and `\}` BEFORE inserting into the ASS text field.
    - `_build_ffmpeg_command(self, audio_path, ass_path, output_path, width, height)`: Returns list of args. Uses `-filter_complex` with `subtitles='escaped_path':fontsdir='escaped_fonts_dir'`. If logo exists, loop it as video input; otherwise use `color=` filter. Always `-c:a copy`, `-preset medium`, `-crf 18`, `-movflags +faststart`, `-shortest`.
    - `create_subtitle_clip(self, audio_path, srt_path, transcript_data, clip_info, output_path, format_type)`: Main single-clip method. Uses `SubtitleGenerator().extract_words_for_clip()` to get clip-relative words from transcript_data using clip_info start/end. Calls normalize, generate ASS, build FFmpeg command, subprocess.run. Returns output path or None on failure. Log warning if FFmpeg stderr contains "substituting font".
    - `create_subtitle_clips(self, clip_paths, srt_paths, transcript_data, best_clips, format_type)`: Batch method iterating over clips. Returns list of output MP4 paths.

    **Important design notes:**
    - `create_subtitle_clip` needs `transcript_data` and `clip_info` (with start_seconds/end_seconds) to extract word-level data. The `srt_path` parameter is accepted for interface compatibility but NOT used for word timing (words come from transcript_data).
    - Use `subprocess.run(list_args)` never `shell=True`.
    - Dimensions for vertical: 720x1280.
    - Output file naming: replace `.wav` with `_subtitle.mp4` in the clip filename.
    - Wrap FFmpeg call in try/except with timeout=300s.
