    **distribute.py — Platform ID capture (ANLYT-01):**
    In `run_distribute()`, after `_upload_youtube()` and `_upload_twitter()` calls complete, build a `platform_ids` dict and write `platform_ids.json` to `episode_output_dir`:
    ```python
    platform_ids = {}
    if youtube_results:
        full_ep = youtube_results.get("full_episode") or {}
        if full_ep.get("video_id"):
            platform_ids["youtube"] = full_ep["video_id"]
    if twitter_results and isinstance(twitter_results, list) and twitter_results:
        if twitter_results[0].get("tweet_id"):
            platform_ids["twitter"] = twitter_results[0]["tweet_id"]
    if platform_ids:
        platform_ids_path = episode_output_dir / "platform_ids.json"
        with open(platform_ids_path, "w", encoding="utf-8") as f:
            json.dump(platform_ids, f, indent=2)
        logger.info("Saved platform IDs: %s", platform_ids_path)
    ```

    **analytics.py — Twitter impression null guard (ANLYT-02):**
    In `fetch_twitter_analytics()`, replace `total_impressions += metrics.get("impression_count", 0)` with null-aware tracking:
    - Track `impression_data_available = False` flag
    - Only set `True` and accumulate when `imp is not None and imp > 0`
    - Return `"impressions": total_impressions if impression_data_available else None`

    In `calculate_engagement_score()`, guard against None impressions:
    - `tw_impressions = tw.get("impressions") or 0` so None becomes 0 for scoring

    **analytics.py — fetch_youtube_analytics video_id param (ANLYT-01):**
    Add optional `video_id: Optional[str] = None` parameter. When provided, skip the `search().list()` call and go directly to `videos().list(id=video_id, part="statistics")`. This reduces quota cost from 100 units to 1 unit per call.

    **analytics.py — Engagement history accumulation (ANLYT-03):**
    Add `append_to_engagement_history()` method to `AnalyticsCollector`:
    - Accepts episode_number, analytics_data dict (with youtube/twitter sub-dicts), platform_ids dict, topics list, post_timestamp str
    - Loads existing `topic_data/engagement_history.json` or starts empty list
    - Builds record with schema from RESEARCH.md (episode_number, collected_at, post_timestamp, topics, youtube metrics, twitter metrics)
    - Twitter impressions stored as-is (None when null)
    - Upserts by episode_number (update existing record, or append new)
    - Writes back with json.dump indent=2

    Add helper `_load_platform_ids(episode_number)` that reads `output/ep_{N}/platform_ids.json`.
